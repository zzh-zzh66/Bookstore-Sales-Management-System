## 三、技术说明

### 3.1 数据文件结构

```
e:\Bookstore-Sales-Management-System\db\
├── lms_schema.sql     # 数据库结构定义 + 基础数据
└── lms_data.sql       # 完整测试数据（本次生成）
```

### 3.2 执行顺序

```bash
# 1. 执行数据库结构
mysql -u root -p < lms_schema.sql

# 2. 执行测试数据
mysql -u root -p < lms_data.sql
```

### 3.3 数据一致性处理

由于 `lms_schema.sql` 末尾已包含部分初始数据（5本书+库存），`lms_data.sql` 采用以下策略：

- **INSERT IGNORE**：跳过已存在的唯一键记录，避免重复插入错误
- **外键关系**：严格遵循 `book → inventory → sale_order → sale_order_item` 的依赖顺序

## 四、问题与解决方案

### 4.1 遇到的问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 初始数据重复 | lms_schema.sql 末尾已有示例数据 | 使用 INSERT IGNORE |
| AUTO_INCREMENT 不重置 | DROP TABLE 不重置自增计数器 | 重建数据库 |
| 外键约束失败 | book_id 硬编码为 1,2,3... 但实际 ID 不同 | 重建数据库确保 ID 从 1 开始 |

### 4.2 最终方案

```sql
-- 完整重建数据库
DROP DATABASE IF EXISTS bookstore_lms;
CREATE DATABASE bookstore_lms;
SOURCE lms_schema.sql;  -- 包含 DROP TABLE，重新创建所有表
SOURCE lms_data.sql;    -- 执行测试数据
```
